## Operating-System Boundary

- Core packages are tested in architecture-native Linux containers and may be
  developed on macOS. Their package contracts do not promise every operating
  system or architecture until a release matrix proves it.
- Chamber's non-privileged contract is portable Rust. The supported production
  containment path is Linux only.
- Cell's launcher, peer host, cgroup, credential, and gVisor claims require the
  measured Linux profile. A macOS unit-test pass is not containment evidence.
- PostgreSQL is an external durable-authority substrate. Cadenza does not claim
  to operate or make PostgreSQL highly available.

## Repository Pins

- Node repositories use `.node-version`.
- Python uses `.python-version` and exact CI versions.
- Elixir uses `.tool-versions` and a locked dependency graph.
- C# uses `global.json` with no SDK roll-forward.
- Rust manifests set `rust-version = "1.97"` and CI selects exact `1.97.0`.
- CI runners use `ubuntu-24.04`; action references are pinned to full commit
  SHAs with explicit permissions.
