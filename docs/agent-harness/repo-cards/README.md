These cards are workspace-level routing aids. Open one before entering a child repo so you know the repo's role, authority boundaries, first files, and basic commands.

If the repo changes Cadenza semantics rather than just consuming them, read [../../cadenza-learning-path.md](../../cadenza-learning-path.md) and [../../vision.md](../../vision.md) first.
