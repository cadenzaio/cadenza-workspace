# cadenza-ui

Role: UI and visualization layer for Cadenza runtime data.

Enter this repo when:

- a task affects the visualization surface, browser interactions, or UI-specific consumer behavior
- service or DB metadata changes need UI propagation

First files:

- `cadenza-ui/README.md`
- `cadenza-ui/package.json`
- `cadenza-ui/src/`
- `cadenza-ui/public/`

Primary commands:

- `cd cadenza-ui && npm run dev`
- `cd cadenza-ui && npm run build`

Contract role:

- Consumer of core, service, and DB contracts

Routing note:

- The current README still reflects starter-kit language, so inspect `src/` directly when the README is too generic.
