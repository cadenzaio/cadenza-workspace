# Local Orchestration Contract V0

## Purpose

This contract defines the language-neutral boundary by which one cell generation owns several chamber identities, publishes bounded runtime truth to them, validates local primitive transport, and replaces chamber generations without exposing topology to authored logic.

It does not define inter-cell transport, placement, actors, persistence, or stem-cell reconciliation.

## Authority

- `cadenza-cell` owns cell-generation identity, host meta authority, chamber registry, projection publication, route validation, and chamber orchestration.
- `cadenza-chamber` owns projection validation/application, chamber-local route state, generated runtime support, primitive ingress, and delegated-execution boundaries.
- `cadenza` owns the task, signal, and inquiry semantics used after route transport accepts a request.

## Canonical Data

All cross-process contracts use bounded canonical JSON:

- printable ASCII object keys sorted by ascending byte value.
- UTF-8 string values containing Unicode scalar values.
- arrays preserve declared order; set-like arrays are sorted and duplicate-free.
- integers are positive and within the interoperable safe range unless explicitly optional.
- floating-point and exponential numbers are forbidden.
- digests use `sha256:` plus 64 lowercase hexadecimal characters.
- identity keys are non-empty lowercase key-like strings using dots as structural separators.

## Cell Generation

A cell process lifetime has one immutable identity:

```json
{
  "cell_generation_key": "cell-generation.local.control.0001",
  "cell_key": "cell.local.control.1",
  "environment_key": "environment.local.primary"
}
```

Every host-authority mutation, projection, acknowledgement, route request, and evidence record binds this identity. `host_meta_authority_revision` starts at `1`, increases by exactly one for each accepted mutation, and has meaning only inside that generation.

Cell disconnect or process loss ends the generation. Frames from an ended or different generation are rejected even when numeric revisions match.

## Host Meta Authority

The cell owns one local truth containing:

- chamber identity, lane, lifecycle, immutable image revision, image epoch, and image digest.
- declared support-slice responsibilities.
- task-entry groups.
- direct signal and target route groups.
- required and acknowledged projection identity per chamber.

The only chamber service states are `starting`, `ready`, `draining`, `failed`, and `stopped`. A chamber is execution-eligible only when it is `ready`, its image generation is current, its exact required projection is acknowledged, and its requested responsibility or route membership is ready.

Host meta authority cannot create semantic graph objects, change environment policy, or redefine primitive meaning.

## Static Runtime Shape

An immutable chamber image declares:

- outbound signal forwarding descriptors.
- target proxy descriptors.
- local intent-to-proxy responder bindings.
- lane and layer constraints.
- exact generated-support artifact identity.

Adding or removing a descriptor changes image revision. Dynamic projection may change candidates and epochs but cannot invent undeclared signals, targets, proxies, responders, or host ports.

## Chamber Authority Projection

A projection is a full replacement snapshot containing:

```json
{
  "cell_generation": {
    "cell_generation_key": "cell-generation.local.control.0001",
    "cell_key": "cell.local.control.1",
    "environment_key": "environment.local.primary"
  },
  "projection_digest": "sha256:<canonical-body-digest>",
  "projection_revision": 1,
  "routes_by_signal": {},
  "routes_by_target": {},
  "self_chamber_key": "chamber.business.source.1",
  "self_image_epoch": 1,
  "source_host_meta_authority_revision": 4,
  "support_slice_keys": []
}
```

`projection_digest` covers canonical JSON without `projection_digest`. Projection revisions increase monotonically per chamber and only when that chamber's derived view changes.

The chamber rejects:

- the wrong environment, cell, generation, self chamber, or image epoch.
- revision `0`, a stale/equal revision after one is applied, or a host revision older than its applied source revision.
- digest mismatch, duplicate route identity, unsorted/duplicate candidates, empty candidates, unknown static descriptor, unknown chamber, or oversized input.

An accepted snapshot atomically replaces all dynamic route state before acknowledgement.

## Projection Acknowledgement

Acknowledgement contains:

```json
{
  "cell_generation_key": "cell-generation.local.control.0001",
  "chamber_key": "chamber.business.source.1",
  "image_epoch": 1,
  "projection_digest": "sha256:<digest>",
  "projection_revision": 1,
  "source_host_meta_authority_revision": 4
}
```

The cell accepts only the exact currently required projection identity. Receiving a projection frame is not sufficient. Eligibility follows successful application and exact acknowledgement.

## Direct Route Groups

Direct route groups expose only opaque route-member candidates:

```json
{
  "candidate_route_member_keys": ["route-member.business.target.1"],
  "route_group_key": "route.target.calculate-total",
  "route_epoch": 1,
  "selection": "single_fixed"
}
```

Selection is `single_fixed` or `round_robin`. `sticky_actor_key` and actor-owner modes are forbidden in V0. A route member resolves inside the cell to exact current distribution authority and may name a local or remote chamber. The chamber receives neither placement topology nor peer credentials.

The source chamber selects from its applied projection. The cell validates the exact source generation, projection revision, route epoch, declared descriptor, layer relationship, selected candidate, target readiness, target image epoch, and deadline. It never substitutes another target.

## Delegation

Delegation names one root `target_key`, selected route member, route epoch,
source projection revision, normal context, explicit causal identity, caller
idempotency, and one absolute deadline.

The private addressed envelope binds `cell_generation_key`,
`source_chamber_key`, `source_image_epoch`, `projection_revision`, `target_key`,
`route_group_key`, `route_epoch`, `selected_route_member_key`, initiating
subject, runtime principal, `trace_key`, `source_graph_execution_key`, the exact
`source_effect_key`, optional `source_inquiry_key`, explicitly named
`idempotency_key`, deadline, and result bound. Generated adapter support selects
the candidate, but the Rust chamber independently revalidates every field
against its immutable support descriptor, current applied projection, trusted
ingress identity, and execution-key namespace before the cell sees the request.
The source cell then creates `distribution_execution_key`; local routing carries
no transport attempt and remote routing creates one `transport_attempt_key` per
connection.

The target starts an ordinary local Cadenza graph at `target_key`. The host distinguishes accepted, graph-started, and graph-concluded boundaries.

The target core owns conclusion semantics. It returns the composed context produced under the [graph conclusion contract](../graph-conclusion/v0.md), or an explicit execution/composition failure. The chamber must not infer a result from exported graph topology or select a terminal context itself.

Source-facing outcomes are:

- `fulfilled`.
- `execution_failed` with `execution_started: true`.
- `retryable_reject` with `execution_started: false` and reason `stale_route` or `target_draining`.
- `denied` with `execution_started: false`.
- `transport_failed` with explicit `execution_started`.

Timeout before graph start is `transport_failed` with `execution_started: false`. Timeout inside a started graph is `execution_failed`. Channel or target loss after start is uncertain and cannot be represented as safely retryable.

A graph composition conflict is `execution_failed` with `execution_started: true`. Conflict diagnostics identify canonical context paths and terminal provenance; raw conflicting values are not part of the ordinary transport contract.

## Signals

Generated forwarding support observes only statically declared outbound signals with current foreign routes. It selects a projected destination and sends one addressed request to the cell.

V0 permits same-lane signal delivery and `business` to `meta_support` delivery. Every other cross-lane direction is rejected. In particular, `meta_support` cannot signal `business`, and `trusted_control` cannot send or receive cross-lane signals. The cell rejects an invalid relationship before accepting route state and revalidates it when publishing projections and interpreting addressed delivery.

The cell validates the exact route and target, then the target chamber injects the signal with an internal no-forward marker. Acceptance acknowledges bounded ingress, not detached-work completion.

The signal envelope binds the same source generation, image, projection, route-group, selected-candidate, caller, correlation, and deadline identities as delegation, replacing `target_key` with `signal_key` and omitting a result bound. The no-forward marker is adapter-owned execution state; it is not a business-context field or authored capability.

## Inquiry

Inquiry does not cross the chamber protocol. An immutable generated proxy task registers as an ordinary local responder for a declared intent and delegates to the declared target task. No `routes_by_intent` projection exists.

Transport outcomes survive the local proxy graph without being inferred from exported graph topology. A pre-target `stale_route` or `target_draining` remains a retryable rejection with `execution_started: false`; target execution and composition failures remain started failures.

## Replacement

Replacement is successor-first:

1. start and activate a successor with a new identity or image epoch.
2. publish successor route membership through a host-authority mutation.
3. require affected source projections to be applied and acknowledged.
4. make the successor eligible.
5. mark the predecessor draining and reject new work.
6. issue drain on the serialized chamber command stream after earlier work has
   returned, then stop or forcibly terminate if a bounded command does not return.

Failure after publication creates a new compensating authority revision and new projections. Prior state and evidence are not rewritten.

The drain acknowledgement is V0 quiescence evidence only because each chamber
control stream is serialized. V0 does not expose an active-execution count and
does not claim concurrent in-flight drain semantics. Concurrent chamber
execution and control multiplexing are deferred until after the first Cadenza
environment is complete.

## Evidence

Canonical ordered evidence records generation, host revision, projection identity, route epoch, image generation, chamber identities, correlation identity, outcome, and started state for high-value transitions.

V0 requires deterministic evidence and proof artifacts but does not define the final durable production evidence store.

## Explicit Non-Goals

- inter-cell transport or distributed envelopes.
- placement units, replicas, capacity, or stem-cell reconciliation.
- actor residency, ownership, hydration, persistence, or actor-aware routes.
- projection deltas or mutable images.
- concurrent chamber execution or control multiplexing.
- generic RPC, generic host calls, ambient transport, or host-selected silent rerouting.
- plugins, secrets, UI, CLI, agents, or memory.
