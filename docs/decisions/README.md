# Decision Log Convention

Approved design-required decisions are recorded here as immutable records.

## Filename convention

Use:

`YYYY-MM-DD-<slug>.md`

Example:

`2026-03-05-actor-session-state-sync-policy.md`

## Required sections

Each decision file should include:

1. Context
2. Decision
3. Consequences
4. Alternatives considered
5. Links (task notes, commits, related docs, or external references)

## Process

- Decision entries are created after explicit design approval.
- Existing decision files are not rewritten for content changes.
- To supersede a decision, create a new file and mark the old one as deprecated in the new entry.
