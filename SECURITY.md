# Security Policy

## Supported Versions

This project is in release-candidate development. Only the latest published release candidate receives security fixes. There is no support SLA.

## Reporting A Vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub private vulnerability reporting for this repository:

https://github.com/cadenzaio/cadenza-workspace/security/advisories/new

Include affected versions, deployment assumptions, reproduction steps, impact, and any suggested mitigation. Maintainers will acknowledge and assess reports when capacity permits; no response or remediation deadline is promised.

## Scope

Security claims are limited to documented contracts, threat models, deployment assumptions, and measured evidence. A release candidate is not a claim of production readiness. Never include secrets, credentials, personal data, or raw business contexts in a report unless essential and explicitly protected.
